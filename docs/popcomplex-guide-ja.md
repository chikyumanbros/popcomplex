# PopComplex ―― ブラウザで動く人工生命シミュレータ

## はじめに

PopComplex は、ブラウザ上で「小さな生き物たち」が生まれ、食べ、増え、進化し、やがて死んでいく様子を眺められる **人工生命（Artificial Life）** シミュレータです。

難しい知識は一切不要です。ページを開くだけで、256×256 マスの世界に色とりどりの生命が湧き上がり、リアルタイムで生態系が組み上がっていきます。

---

## 何が面白いのか

### 1. 誰もデザインしていない「生き物」が勝手に現れる

PopComplex の生き物には「こう動け」というプログラムを人間が書いていません。最初に植え付けられるのは、ごく単純なルール表と小さなニューラルネットワーク（脳）だけ。世代を重ねるうちにコピーミス（突然変異）が蓄積し、**環境に適応したものだけが残る** ―― ダーウィン的な自然選択が、ブラウザの中で本当に起きます。

### 2. エネルギーは有限 ―― 「閉じた宇宙」の生存競争

この世界の総エネルギーは完全に一定です。食べれば環境からエネルギーが減り、死ねば環境に戻る。誰かが豊かになれば、別の誰かが飢える。現実の生態系と同じ「ゼロサム」の厳しさが、単純な物理法則だけで自然に立ち現れます。

### 3. 「脳」と「遺伝子」がひとつのテープに載っている

各生物の全情報は **256バイトのテープ** 1本に詰まっています。行動ルール、身体の設計図、脳の重み ―― すべてが同じバイト列の中にあり、突然変異も劣化も同じ仕組みで起きます。これは生物学で言う「DNA が脳の配線図も含む」ことの極端な抽象化です。

### 4. WebGPU でヌルヌル動く

シミュレーションは CPU が担当し、その結果を GPU（WebGPU）がリアルタイム描画します。数千の生物がうごめく 65,536 マスの世界が、モダンブラウザだけで滑らかに動きます。

## 世界のしくみ

### グリッド（世界地図）

世界は **256×256 = 65,536マス** の格子です。各マスは次の情報を持ちます。

| 情報 | 説明 |
|------|------|
| 環境エネルギー | そのマスに漂っている「栄養」。生物がいなくても拡散で隣へ流れる |
| 細胞エネルギー | 生物の細胞が蓄えているエネルギー |
| 胃袋（stomach） | 食べたけどまだ消化していないエネルギー |
| モルフォゲン A/B | 生物が出す化学シグナル（体の形づくりに使う） |
| マーカー | 食事・消化・信号・移動の「痕跡」。可視化モードで色として見える |

### エネルギー保存則（閉じた宇宙）

```
環境エネルギーの総和 ＋ 全細胞エネルギーの総和 ＋ 全胃袋の総和 ＝ 一定
```

この等式は **毎ティック** 強制的に補正されます。エネルギーが消えたり湧いたりすることは原理的にありません。これが PopComplex の「物理法則」の根幹です。

### シミュレーション 1 ティックの流れ（代謝まわりの交通図）

細部は省き、**エネルギーと行動**に関係する順だけ示します（実装は `simulationTick`）。

1. **テープから脳へ** — NN の重みをテープから読み込む。  
2. **脳を更新** — 個体全体の統計から入力を作り、気分（4 出力）を計算する。  
3. **`evaluate`** — 環境の拡散、受動吸収、マーカー、**ルール表の走査とアクション**（EAT、GIVE、移動など）、シグナル伝播など。ここで **環境→胃** の取り込みや **細胞間の直結** が主に起きる。  
4. **`digestPhase`** — **胃袋→細胞エネルギー** への変換（熱の一部は環境へ）。**ここだけ**が胃の本線の消化。  
5. **維持代謝** — 密度などに応じて細胞エネルギーが環境へ戻る。  
6. **個体オーバーヘッド** — 系統あたりの固定負担を細胞で按分。  
7. **死体処理・分裂・予算補正** など。

### エネルギーの上位法則（覚えるならこの3つ）

1. **外界から生物へ** — 環境の栄養が細胞に入る本線は **まず胃袋**（受動吸収・EAT）。  
2. **胃から細胞へ** — 胃の中身が細胞エネルギーになるのは **`digestPhase` のみ**。ルールの **DIGEST** はこのフェーズを **強めるブースト** を積むだけで、即座にエネルギーを動かしません。  
3. **同じ個体内の直結** — **GIVE / TAKE** は近傍の **細胞エネルギー** を直接やりとりします（説明上は **結果供与**／**結果搾取** と呼ぶと分かりやすいです）。別個体とのやりとりは条件付きで、捕食などは **胃経由** になる経路もあります。

## 生き物のしくみ

### テープ ―― 256バイトの「全存在」

各生物は **テープ（Tape）** と呼ばれる 256 バイトのデータを持っています。人間でいえば DNA に相当しますが、脳の配線図まで含んでいる点がユニークです。

```
テープの構造（256バイト）

┌─────────────┬──────────────┬──────────────────┬──────────────┐
│  0～31      │  32～63      │  64～127         │  128～255    │
│  データ領域  │  CA帯        │  ルール表        │  脳の重み     │
│  (身体の設計)│  (免疫キー等) │  (16個の行動規則) │  (64パラメータ)│
└─────────────┴──────────────┴──────────────────┴──────────────┘
```

- **データ領域（0～31）**: 体の最大サイズ、食事の強さ、分裂のコストなど、基本的なパラメータ。系統の「色」もここに入っています。
- **CA帯（32～63）**: 神経の不応期、エネルギー容量の設計、そして **複製キー**（子供を作るときの「鍵」）。
- **ルール表（64～127）**: 16 個の「もし○○なら××せよ」という条件→行動ペア。これが生物の具体的な振る舞いを決めます。
- **脳の重み（128～255）**: 小さなニューラルネットワーク（8入力 → 4隠れ → 4出力）の全パラメータ。

### 劣化（degradation）

テープには、データとは別に **劣化トラック** がもう 256 バイトあります。これは「どれだけ傷んでいるか」の記録です。生きている間にテープは少しずつ壊れていき（ビット反転）、とくにエネルギーが低いとき・高齢のときに壊れやすくなります。

子供には **劣化は引き継がれません**。生まれたての子供のテープは真っさらです。ただし、親の劣化が激しいとコピーミスが増え、場合によっては **死産** になります。

### 脳 ―― 4つの「気分」

脳は **8つの感覚** を入力として受け取り、**4つの気分（mood）** を出力します。

**入力（8つ）:**
1. 平均エネルギー（自分の細胞がどれくらい元気か）
2. 平均胃袋（消化待ちの食べ物がどれくらいあるか）
3. 周囲の環境エネルギー（栄養が豊富な場所にいるか）
4. 自分の大きさ（何マス占めているか）
5. 境界比率（自分の表面のうち外に面している割合）
6. 異種接触率（隣にいる他の生物の多さ）
7. マーカー優勢度（自分のテリトリーの化学的な優位性）
8. 環境勾配（近くに栄養が多い方向があるか）

**出力（4つの「気分」）:**
- **EAT**: 「食べたい」
- **GROW**: 「成長・繁殖したい」
- **MOVE**: 「移動したい」
- **CONSERVE**: 「じっとしてエネルギーを節約したい」

4つの出力は確率分布になっていて、一番強い気分が「今の支配的な行動方針」になります。この気分がルール表の条件判定にも影響を与えるため、**脳とルール表は連携して動きます**。

### ルール表 ―― 16 個の「if-then」

ルール表の各行は 4 バイトで、次の形をしています。

```
条件フラグ ｜ しきい値 ｜ アクション ｜ パラメータ
```

例えば「環境エネルギーが 1 より大きければ → 食べろ（強さ 140）」のように読みます。

**使えるアクション一覧:**

| アクション | やること |
|-----------|---------|
| EAT | 環境からエネルギーを胃袋に取り込む |
| DIGEST | **消化ブースト** — そのティックの `digestPhase` での胃→細胞の変換を強める（即時には動かない） |
| DIV | 隣の空きマスに新しい細胞を作る（体を大きくする） |
| GIVE | **結果供与** — 近傍の細胞の細胞エネルギーを増やす（仲間に分け与える／信頼できる異種への経路あり） |
| TAKE | **結果搾取** — 近傍から細胞エネルギーを自分へ（仲間からもらう；異種は現状ほぼ同種のみ） |
| MOVE | 体全体を隣のマスに動かす |
| REPRODUCE | 子供を産む（テープをコピーして新しい個体を作る） |
| ABSORB | 他種の細胞からエネルギーを奪う（捕食） |
| FIRE / SIG | 神経信号を発火させる |
| EMIT | モルフォゲン（化学物質）を放出する |
| REPAIR | テープの劣化を修復する（免疫のようなもの） |
| SPILL | 胃袋の中身を環境にこぼす |
| JAM | 異種との結合を一時的に遮断する防御壁（この境界では **緩和型 ABSORB**・**信頼に基づく異種 GIVE**・**外国キンによる修復の助け**・**接触経由の遺伝子移動** などが止まる。捕食による窃取は別経路） |
| NOP | 何もしない（壊れた行もここに落ちる） |

ルール表のオペコード（アクション番号）が壊れると、その行は **NOP（何もしない）** として扱われます。つまり、**テープが劣化すると行動レパートリーが狭まっていく** のです。

**走査のしかた**: 16 行を **毎ティック必ず 0 番から** ではなく、`(ティック + 細胞位置 + 個体 ID)` でずらした行から **一周** します。条件が合えば **複数行が同じティックで実行されうること** もあります（MOVE が成功した細胞は打ち切り）。

**chain（連鎖）ビット**: 条件フラグの **bit 6** が立っている行は、条件が真でも **アクションを実行せず**、次の行へ「つなぐ」だけです。直後の行は、その連鎖が生きているときだけ追加条件として評価されます。**空間上のつながり**ではなく、**ルール同士の論理のつながり**です。

## ライフサイクル

### 誕生

最初の生物は **プロトタイプテープ** から生まれます。起動時に 3 か所に 10 細胞ずつ植え付けられ、そこから世界が始まります。同じテープのコピーなので最初は同じ挙動ですが、すぐに突然変異で分岐していきます。

### 成長

EAT で環境から栄養を取り、DIGEST で消化し、DIV で細胞を増やします。大きくなるほど代謝コストが上がるため、際限なく大きくはなれません。

### 繁殖

REPRODUCE が発動すると、テープが **コピー（転写）** されます。このとき：
- ビットが反転する（点突然変異）
- バイトの位置が入れ替わる（転座）
- 隣のバイトを誤読する

といったコピーエラーが起き、**子は親と微妙に違う** 個体になります。

もし複製キーが大きく壊れていると、子供は生まれる前に死んでしまいます（死産）。これが「遺伝子の品質管理」として機能します。

### 分裂

大きな体が途中でちぎれると、**連結していない断片が別個体として独立** します。これも新しい系統を生む重要な経路です。

### 死

細胞のエネルギーがゼロになると、確率的にその細胞は溶解します。溶解時に胃袋に残っていた未消化のエネルギーは環境に返されます。すべての細胞を失った個体は消滅します。

## 進化が起きるメカニズム

PopComplex では、進化の三要素がすべて揃っています。

1. **変異**: テープのコピーエラー（突然変異）と、生きている間のテープ劣化
2. **選択**: エネルギーが有限なので、効率よく食べ・消化し・無駄を減らせる個体だけが生き残る
3. **遺伝**: テープが親から子にコピーされる

さらに面白いのは：

- **水平遺伝子移動**: 異種と接触したとき、まれにルールの一部が相手に取り込まれることがあります（ABSORB 経由）。
- **修復（REPAIR）**: 仲間が多いほど修復の成功率が上がります。つまり **集団でいることに免疫的な利点** があります。
- **社会的合意**: 同じ個体の隣接細胞は、シグナルマーカーが自然に揃っていきます（ゴシップ的な局所平均化）。合意度が高いと修復も効きやすくなります。

## 見どころ ―― 観察のヒント

### 系統の興亡を追う

Ecology Chart を眺めていると、ある系統が急に広がり、やがて衰退して別の系統に取って代わられる ―― という歴史的なドラマが見えます。これは現実の生態学で言う「適応放散」や「競争排除」に似た現象です。

### 形態の多様性

- **大きな塊**: 内部の細胞は消化効率が高く、代謝も安い。ただし端の細胞しか食べられない。
- **細長いフィラメント**: 表面積が大きく食事効率は良いが、ちぎれやすく代謝コストが高い。
- **単細胞**: 系統維持のオーバーヘッドが重いが、機動力がある。

### 「気分」の偏り

Default 表示でよく見ると、色の違いだけでなく「動き方」にも個性があります。ずっと同じ場所にいる保守的な個体、栄養を求めて動き回る探索者、ひたすら食べて太る貯蔵型 ―― これらは脳（NN）の重みの違いから生まれています。

### テープの「老化」

高齢の個体は、ルール表のオペコードが壊れて NOP だらけになっていることがあります。すると行動パターンが単純化し、「生き残ってはいるが、もはや複雑な振る舞いはできない」という老化現象が見られます。壊れた行には微小なエネルギー税がかかるため、劣化は緩やかに不利に働きます。

## 技術的なポイント（もう少し知りたい人向け）

### なぜ CPU シミュレーション？

シミュレーションの各ステップでは「ある個体の行動が別の個体に即座に影響する」ような逐次依存が多いため、GPU の並列計算とは相性が悪い部分があります。そのため、**正確さが必要なシミュレーションは CPU、描画だけを GPU** という分担になっています。

### エネルギー会計の厳密さ

あらゆるアクションで「失われたエネルギーは必ずどこかに行く」ように設計されています。食べれば環境から減り、代謝熱は環境に戻り、死んだ細胞のエネルギーも環境に返る。毎ティック末に残差を補正するセーフティネットもあります。

### 種のないシード

`seed` はシミュレーション中の乱数列を決める数値です。同じ seed なら同じ歴史が再現されます。「この seed だと面白い生態系になる」という発見は、まさに宇宙の初期条件を探すような楽しさがあります。

## まとめ

PopComplex は、**物理法則だけを定義して、そこに「生命」を放り込んだらどうなるか** を見る実験場です。

- ルールは単純。でもそこから生まれる挙動は複雑。
- 設計者が意図しなかった「生態系」が自然に立ち上がる。
- 同じプログラムでも seed が違えば、まったく違う世界が広がる。
